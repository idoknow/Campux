# 运营工作台

校园墙运营者主要使用“稿件”“管理”“统计”三个区域。日常工作不需要进入系统运维面板，除非你同时负责实例级维护。

如果你是从官方管理端第一次注册的运营管理员，先按 [自助开墙流程](/operator/self-service-onboarding) 创建校园墙、接入 NapCat 和配置发布目标；完成后再回到这篇文档处理日常运营。

投稿页是用户和运营者都会看到的入口；审核员和管理员会额外看到审核、管理和统计能力。

![投稿页面](/screenshots/post-compose.png)

审核工作台集中显示待处理稿件、投稿人信息、图片数量、匿名状态和审核操作。

![审核稿件列表](/screenshots/review-board.png)

## 日常工作顺序

1. 在“稿件 / 审核”里处理待审核稿件。
2. 对可发布内容点击“通过”，对不适合发布的内容填写拒绝理由。
3. 在“管理 / 发布”里查看发布目标、cookies 状态和失败日志。
4. 在“管理 / 机器人”里检查协议端是否在线。
5. 在“统计”里复盘投稿量、通过率和发布质量。

服务页用于放置修改密码、投稿规则和运营方配置的入口。

![服务入口](/screenshots/services.png)

## 页面权限

| 页面 | 用户 | 审核员 | 管理员 |
| --- | --- | --- | --- |
| 投稿 | 可用 | 可用 | 可用 |
| 稿件：你的稿件 | 可用 | 可用 | 可用 |
| 稿件：审核 | 不可用 | 可用 | 可用 |
| 服务 | 可用 | 可用 | 可用 |
| 管理 | 不可用 | 不可用 | 可用 |
| 统计 | 不可用 | 可用 | 可用 |

## 运营边界

运营者可以管理当前校园墙内部事务，例如公告、投稿规则、成员、封禁、机器人和发布目标。以下能力属于系统维护者：

- 创建、暂停、归档租户。运营管理员只能处理自己负责的校园墙，系统运维可以处理全部租户。
- 配置租户专属 host。
- 给任意账号添加运营管理员或系统运维身份。
- 查看全局用户和全局审计日志。

## 推荐配置检查表

- 校园墙名称、品牌名和公告是否正确。
- 投稿规则是否清楚，是否适合在移动端阅读。
- 至少一个机器人已连接 OneBot。
- 至少一个发布目标启用且 cookies 可用。
- 审核群能收到新稿件、通过、拒绝、取消和发布失败通知。
- 管理员和审核员账号已授权到当前校园墙。
