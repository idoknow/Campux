# 成员、封禁与配置

管理员在“管理”页维护当前校园墙的成员、封禁、元数据、机器人和发布目标。

## 成员管理

成员管理用于把全局账号授权到当前校园墙，并设置租户内身份：

![成员管理](/screenshots/admin-users.png)

| 身份 | 能力 |
| --- | --- |
| 用户 | 投稿、查看自己的稿件、取消审核中的稿件 |
| 审核员 | 审核稿件、查看统计 |
| 管理员 | 管理成员、封禁、元数据、机器人、发布目标 |

用户数量较多时，请使用搜索和分页，不要通过下拉列表查找账号。

## 封禁管理

封禁用于阻止账号使用当前校园墙。被封禁账号进入该租户后只会看到封禁信息，不能访问任何页面。

![封禁管理](/screenshots/admin-bans.png)

封禁记录应包含：

- 用户 QQ 或昵称。
- 封禁原因。
- 开始时间和结束时间。
- 是否仍在生效。

结束时间会显示完整年份，适合长期封禁。

## 元数据

元数据属于租户管理员维护范围，包括：

![元数据管理](/screenshots/admin-metadata.png)

- 校园墙名称。
- 前台品牌名。
- slug。
- 主题色。
- 公告。
- 投稿规则。
- 服务入口。

专属 host 不在这里配置，属于系统运维统一管理。

## 服务入口

服务页会固定提供“投稿规则”和“修改密码”，即使配置了自定义服务也不会把它们挤掉。自定义服务适合放：

![服务入口](/screenshots/services.png)

- 校园互助表单。
- 社群链接。
- 招新入口。
- 常见问题说明。
