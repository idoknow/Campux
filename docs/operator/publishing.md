# 发布目标与 QZone

发布目标描述稿件通过审核后要发布到哪里。一个校园墙可以有多个发布目标，例如多个 QQ 空间墙号。

![发布管理](/screenshots/publishing.png)

发布目标、cookies 状态和最近发布日志在同一个页面内处理；日志按稿件分组，便于追踪某条稿件到底发到了哪些墙号。

## 发布目标字段

| 字段 | 说明 |
| --- | --- |
| 机器人 | 负责执行发布的 Bot |
| 目标名称 | 运营后台展示名 |
| 风控间隔 | 连续发布之间的最小等待秒数 |
| 必需发布目标 | 失败时是否阻塞稿件进入已发布状态 |
| cookies 刷新方式 | 协议自动获取或扫码登录 |
| 文字模板 | 发布说说时拼接到图片或正文链接周围的文案 |

默认风控间隔为 10 秒。实际生产环境可按墙号风控情况调大。

## cookies 刷新

Campux 支持两种刷新方式：

1. **协议自动获取**：调用 OneBot 协议端能力获取 QZone cookies。
2. **扫码登录**：在网页或审核群中启动扫码登录流程，由运营者用 QQ 扫码。

不是所有协议端都支持自动获取 cookies，因此建议至少确认扫码登录流程可用。

## cookies 健康检查

发布页显示的不是网络延迟，而是 cookies 是否可用。系统会记录：

- 最近检测时间。
- cookies 状态。
- 检测失败信息。
- 最近刷新时间。

当 cookies 失效时，Campux 会在审核群提示运营者重新登录。

## 发布日志

发布日志按稿件分组展示，运营者可以看到：

- 稿件编号和正文摘要。
- 每个发布目标的状态。
- 最近一次 HTTP 返回值或错误信息。
- 是否可以重试。

如果网页显示“已发布”但 QQ 空间没有内容，应优先展开对应稿件的发布详情，对照 QZone HTTP 返回值判断是否真的发布成功。

## 文案模板

每个机器人可以有自己的说说文案模板。模板支持：

- 固定前缀。
- 固定后缀。
- 是否包含稿件编号。
- 是否在非匿名时带 `@用户`。
- 是否提取正文链接。

正文图片由渲染流程生成，发布说说时不应把完整正文重复摆进配文里。
