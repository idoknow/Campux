# 运维面板

系统运维面板独立于具体校园墙。它用于管理实例级资源，而不是维护某个校园墙的公告或投稿规则。

![系统运维面板](/screenshots/ops-panel.png)

## 运维面板负责什么

- 创建、暂停、恢复、归档租户。
- 查看每个租户的用户数、墙号数和稿件数。
- 配置租户专属 host。
- 查看全局用户并调整系统级或租户级身份。
- 查看全局审计日志。
- 快速定位 Bot 与发布目标状态。

## 不应该放在运维面板里的配置

以下配置属于租户管理员，不属于系统运维：

- 校园墙名称。
- 前台品牌名。
- slug。
- 主题色。
- 公告。
- 投稿规则。
- 服务入口。
- 发布目标文案模板。

系统运维只处理租户生命周期和跨租户边界，避免把某个校园墙的日常运营配置混到系统级面板。

## 运维账号

系统运维身份是全局账号权限。一个账号可以同时是：

- 系统运维。
- 某些校园墙的管理员。
- 另一些校园墙的审核员或普通用户。

进入校园墙工作区和进入运维面板是两个不同入口。多租户账号登录后会先进入选择页。
