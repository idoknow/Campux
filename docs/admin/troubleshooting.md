# 故障排查

## 登录后进不了校园墙

检查：

1. 账号是否有租户成员身份。
2. 如果访问的是专属 host，账号是否属于该 host 对应租户。
3. 账号是否被当前租户封禁。
4. 是否处于首次登录强制修改密码状态。

## OneBot 连不上

检查：

1. 连接 URL 是否完整包含 `bot_id` 和 `token`。
2. 协议端是否配置 WebSocket 反向连接。
3. 服务器 `/onebot/v11/ws` 是否可访问。
4. Bot 是否在 Campux 中启用。
5. 最近事件和最近连接时间是否更新。

## 审核群命令无效

检查：

1. 消息是否 at 了正确 Bot。
2. 命令格式是否为 `#通过 <稿件编号>` 或 `#拒绝 <理由> <稿件编号>`。
3. 审核群号是否配置在对应机器人上。
4. Bot 的 `self_id` 是否和 Campux 记录的 Bot QQ 一致。

## 显示已发布但空间看不到

不要只看稿件状态。请到“管理 / 发布”展开该稿件的发布详情，核对：

- 每个发布目标是否成功。
- QZone HTTP 返回值。
- 图片上传是否成功。
- 是否只是非必需发布目标失败。
- 是否触发风控或 cookies 失效。

## 图片裂了

检查：

1. S3 对象是否存在。
2. `S3_PUBLIC_BASE_URL` 是否能从浏览器访问。
3. MinIO bucket 是否允许读取。
4. 渲染图是否成功写入对象存储。

## 启动失败

常见原因：

- PostgreSQL 不可连接。
- migration 执行失败。
- 生产环境缺少 `CAMPUX_BOT_SESSION_SECRET`。
- S3 配置错误。
- 端口 `8989` 被占用。
