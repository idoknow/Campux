# 租户生命周期

租户代表一个校园墙。系统运维负责创建租户、配置专属 host，以及在必要时暂停或归档。

![系统运维面板](/screenshots/ops-panel.png)

## 创建租户

创建租户时需要：

- 校园墙名称。
- slug。
- 主题色。
- 可选专属 host。
- 可选初始 Bot QQ。

创建租户后，所有系统运维会自动加入该租户，避免新租户创建后无人能进入管理。

## 状态

| 状态 | 说明 |
| --- | --- |
| 运行中 | 用户可以进入并正常投稿、审核、发布 |
| 暂停 | 暂时阻止用户使用，保留数据 |
| 归档 | 长期下线，保留历史记录 |

暂停和归档都属于高影响操作，执行前应确认该校园墙运营者已经知情。

## 专属 host

专属 host 用于把某个域名固定到某个校园墙：

```text
wall.example.com
```

如果用户从这个域名访问 Campux，登录后即使账号属于多个校园墙，也会直接进入该 host 对应的校园墙，不展示选择页。

没有专属 host 的多租户账号会进入选择页；系统运维账号也可以从这里直接进入运维面板。

![校园墙选择页](/screenshots/tenant-selection.png)

适用场景：

- 每个校园墙独立域名。
- 学校公众号或群公告只放一个固定入口。
- 避免普通用户看到“租户”概念。

不要把同一个 host 配给多个租户。
